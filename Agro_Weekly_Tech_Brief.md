# Farmhannong Agro Weekly — 발표자료 제작용 기술 자료

> **이 문서의 용도**
> 이 문서는 "Farmhannong Agro Weekly" 사내 사이트를 회사 동료/임원 앞에서 발표하기 위한 **PPT를 생성형 AI로 제작**할 때, AI에게 그대로 전달하는 입력 자료입니다.
> 사이트의 **필요성 → 기능 → 기술 구조 → 보안 → 운영 → 주의사항**까지 발표에 필요한 모든 내용을 담았습니다.
> 문서 맨 끝의 **"AI에게 주는 PPT 제작 지시"** 섹션을 함께 전달하면 슬라이드 구성·디자인 톤까지 맞춰 만들 수 있습니다.

## 0. 한 줄 요약

> **해외 작물보호제(농약) 시장 동향을 매주 자동으로 수집·번역·정리해 카드뉴스로 보여주는, 검색·개인화·보안이 가능한 사내 정보 포털.** 사람이 매주 약 2시간 들이던 번역·정리·배포 작업을 **AI가 무인으로 대체**하고, 2020년부터의 동향을 **검색 가능한 자산으로 축적**합니다.

- 서비스명: **Farmhannong Agro Weekly** (농협(팜한농) 해외 작물보호 사업용 주간 동향 카드뉴스)
- 형태: 웹사이트 + 모바일 앱(설치형 PWA)
- 접속: 입장 코드 기반 보안 접속 (사내 한정)
- 제작 방식: **생성형 AI(Claude)** 로 기획·개발·운영 자동화까지 전 과정 구축 → **사내 AI 활용 베스트 프랙티스 사례**

## 1. 왜 필요한가 (배경 · 문제 · 해결)

### 1-1. 배경
- 팜한농의 해외 작물보호 사업은 미국·브라질·호주·EU·인도 등 **해외 시장의 농약 등록/출시/규제/시세/M&A 동향**을 빠르게 파악하는 것이 경쟁력의 핵심.
- 관련 정보는 해외 매체·기관에 **영문으로, 산발적으로** 흩어져 있어 담당자가 매주 직접 찾아 읽고 번역·정리해 왔음.

### 1-2. 기존 방식의 문제점
| 문제 | 내용 |
|---|---|
| **시간 소모** | 번역 + 정리 + 배포에 **매주 약 2시간** 소요 |
| **형식 들쭉날쭉** | 작성자마다 정리 형식·품질이 달라 가독성·신뢰도 편차 |
| **휘발성** | 메일·문서로 배포되고 끝 → **검색·재활용 불가**, 지식이 축적되지 않음 |
| **전문용어 장벽** | 잡초·해충·병해·농약 성분의 영문/학명을 매번 따로 찾아야 함 |
| **접근성** | 보고 싶을 때 한곳에서 모아 보거나, 관심 분야만 골라 보기 어려움 |

### 1-3. 해결 — 이 사이트가 하는 일
- **자동화**: AI가 매주 해외 기사를 검색·번역·요약해 **표준 카드 형식**으로 생성·배포 (사람 개입 0).
- **축적**: 모든 주차가 한 곳에 쌓여 **연도·국가·기업·제품군·키워드로 검색** 가능한 데이터베이스가 됨.
- **이해 보조**: 전문용어집이 자동으로 카드에 한국어 각주를 달아줌(그리고 매주 성장).
- **개인화**: 로그인 없이도 북마크·관심 분야 팔로우가 사용자별로 동작.
- **어디서나**: 데스크탑·모바일·설치형 앱으로 접근, 입장 코드로 사내만 접근.

## 3. 기능 전체 (상세)

### 3-1. 카드뉴스 (핵심)
- 한 건 = **카드 한 장**: 분류 배지(등록/개발/영업), 출처 배지(자체검색/리포트), 국가, 발행일, 제목, 제품군 태그, **3불릿 요약 본문**, 출처 링크.
- 말투는 기사체 **"~했음/~함"** 형태로 통일(간결·일관).
- 분류 3종: **등록(REG)** · **개발(DEV)** · **영업/시장(SALES)**.
- 제품군: 살충제·살균제·제초제·생물농약·유기농업자재·비료·종자·기타.
- 각 카드의 **"원문·상세 보기"** → 영문 원문 제목/본문, 키워드 등 상세 팝업.

### 3-2. 강력한 검색·필터
- 통합 키워드 검색 + 다중 선택 필터: **연도 · 발행일 · 분류 · 제품군 · 기업 · 국가**.
- 적용된 필터를 칩으로 표시하고 개별 해제 가능, "초기화" 한 번에 리셋.

### 3-3. 전문용어집 (Glossary) — 자동 성장
- 카드 본문의 영문 잡초·해충·병해·농약 성분 용어에 **자동으로 "* 한국어(학명)" 각주**를 표기.
- 헤더 **"용어집"** 버튼 → 카테고리별·검색 가능한 사전 모달.
- **매주 새 카드가 생성될 때 새 용어가 사전에 자동 추가** → 쓸수록 똑똑해짐.

### 3-4. 개인화 (로그인 없이)
- **북마크**: 관심 카드를 저장하고 "북마크 보기"로 모아 보기.
- **관심(팔로우)**: 관심 **국가·제품군**을 지정하면 해당 조건 카드만 모아 보기.
- 개인화 정보는 **각 사용자의 브라우저에 저장**되어, 같은 사이트를 여러 명이 로그인 없이 써도 서로 섞이지 않음. (원리: 브라우저 localStorage)

### 3-5. 트렌드·인사이트 (데이터 시각화)
- 헤더 **"트렌드"** → KPI 카드(전기 대비 증감), 분류/국가/제품군별 차트, 키워드 추이.
- 차트 요소 클릭 시 해당 조건으로 **드릴다운**(바로 필터링).

### 3-6. F-MBR 월간 보고
- 헤더 **"F-MBR"** → 월간 보고 형식으로 정리해 보는 기능.

### 3-7. 리포트(PDF) 업로드 → 카드화  *(관리자 전용)*
- 기존 레퍼런스 리포트(PDF)를 업로드하면 브라우저(pdf.js)가 본문을 추출.
- **업로드 시 "어느 주차에 넣을지"를 사용자가 선택**.
- 추출분은 다음 주간 생성 단계에서 AI가 **자체 검색 20건과 동일한 형식의 카드(최대 20건, "리포트" 배지)** 로 번역·정리해 합침.
- 어떤 파일이 언제·어느 주차로 반영 예정인지 **대기열 팝업**에서 확인.
- ⚠️ 업로드 즉시 번역되는 것이 아니라 **주간 생성(AI) 단계에서 처리**됨(아래 6장 참고).

### 3-8. 출력 / 인쇄
- 헤더 **"출력"** → 현재 검색·필터 결과를 인쇄/출력용으로 정리.

### 3-9. 모바일 전용 UX
- 모바일에서는 **제목 리스트 → 항목 탭 → 해당 카드 상세(전체화면)** 방식으로 최적화.
- 반응형 레이아웃·중앙 정렬 정돈.

### 3-10. 앱(PWA) 설치
- 전용 아이콘·매니페스트 제공 → 휴대폰/PC에 **앱처럼 설치**해 홈 화면에서 바로 실행.

## 4. 기술 구조 (아키텍처)

> 발표에서는 "가볍고 안전하며 운영비가 들지 않는 구조"라는 메시지로 요약 가능.

- **단일 정적 사이트**: 별도 서버·DB 없이 동작하는 정적 웹(HTML/CSS/JavaScript). 서버 운영·유지보수 부담 없음.
- **호스팅**: **GitHub Pages**(무료). 코드 푸시 → 자동 배포.
- **데이터**: 모든 카드 데이터는 사이트 내부에 내장(외부 DB 불필요).
- **PDF 처리**: 브라우저 내 pdf.js로 업로드 PDF의 텍스트를 추출(서버 전송 없음).
- **앱화**: PWA(매니페스트 + 아이콘)로 설치형 앱 지원.
- **제작·운영 자동화**: 생성형 AI(Claude)가 기획·코딩·주간 콘텐츠 생성·배포·검증을 수행.

### 공개 저장소에 들어가는 것 (보안 경계)
- **공개되는 것**: 입장 화면(로더), **암호화된 데이터(payload.enc)**, 아이콘 등 비민감 자산뿐.
- **공개되지 않는 것**: 원본 콘텐츠 파일과 입장 코드 목록은 **로컬에만 보관**(저장소에 올리지 않음).

## 5. 보안 체계

> 핵심 메시지: **"사내 정보가 인터넷 공개 저장소에 올라가도, 코드를 모르면 내용을 볼 수 없다."**

- **입장 코드 게이트 + 강력한 암호화**: 콘텐츠 전체를 **AES-256-GCM**으로 암호화해 배포. 방문자가 올바른 **입장 코드**를 입력해야 브라우저에서 복호화되어 열림.
  - 코드 → 키 변환에 **PBKDF2(SHA-256, 15만 회 반복)** 적용 → 무차별 대입 저항.
  - 코드 자체는 소스에 평문으로 저장하지 않음.
- **인원별 코드 10개 + 관리자 코드**: 사람마다 코드를 배포해 접근 통제.
- **재방문 편의**: 한 번 입장한 기기는 다음 접속 시 코드가 **미리 채워진 상태**로 입장 화면이 떠 한 번에 입장(직전 개선 사항).
- **권한 분리**: 리포트(PDF) 업로드 기능은 **관리자 코드 보유자만** 사용.

## 6. 운영 — 매주 어떻게 돌아가나

- **주간 자동 생성**: 매주 월요일 오전 9시경(KST) 예약 작업이 실행되어,
  1) 지난 한 주 해외 작물보호 기사를 검색,
  2) **실제 출처가 있는 기사만** 20건 카드로 번역·요약,
  3) 리포트가 있으면 최대 20건 추가,
  4) 새 전문용어를 용어집에 추가,
  5) 재암호화 → 배포 → **라이브 검증**,
  6) **완료 알림**.
- **무인 운영**: 콘텐츠 생성부터 배포·검증까지 사람 승인 없이 진행.
- ⚠️ **수신자 메일 발송만은 수동**: 외부로 나가는 행위이므로 관리자가 직접 실행(안전장치).

## 7. 주의사항 · 한계 (정직하게)

> 발표에서 "한계도 알고 통제하고 있다"는 신뢰를 주는 슬라이드로 활용.

1. **개인화는 기기·브라우저 단위**: 북마크·관심 설정은 그 브라우저에 저장됨. **다른 기기와 동기화되지 않으며**, 브라우저 데이터를 지우면 사라짐(계정 로그인 방식이 아님).
2. **배포 때마다 코드 재입력 가능성**: 콘텐츠를 갱신/재암호화하면 보안 키가 바뀌어, 기존 기기도 **다음 접속 시 코드를 한 번 더 입력**해야 함. → **코드 미리 채움**으로 한 번 탭이면 입장하도록 완화.
3. **코드 회수의 특성**: 특정 코드를 막아도, 이미 들어와 데이터를 받은 기기는 다음 재암호화 전까지 열람 가능(완전 즉시 차단은 재암호화 시 적용).
4. **업로드 ≠ 즉시 번역**: 리포트 PDF를 올려도 즉시 카드가 되는 게 아니라 **주간 생성(AI) 단계에서 처리**됨.
5. **자동 실행 전제 조건**: 무인 주간 생성은 관리자 PC에서 **AI 실행 환경(Claude 앱)이 켜져 있어야** 동작(맥 전원만으로는 불충분).
6. **출처 신뢰성 원칙**: 모든 카드는 **실제 존재하는 기사·URL** 기반으로만 생성하며 출처를 지어내지 않음.
7. **보안 경계 준수**: 원본 콘텐츠·입장 코드 목록은 **절대 공개 저장소에 올리지 않음**(자동화 규칙으로 강제).

## 8. AI 활용 사례로서의 의의 (사내 베스트 프랙티스 포인트)

- **기획→개발→운영→유지보수**의 전 주기를 생성형 AI로 구축한 실제 업무 적용 사례.
- 반복적·고숙련 단순작업(번역·정리·배포)을 **무인 자동화**해 담당자를 고부가 업무에 재배치.
- 휘발되던 정보를 **검색 가능한 자산**으로 전환(6년치 5천여 건 축적).
- 추가 인력·서버·라이선스 비용 없이(호스팅 0원) 운영 → **저비용 고효율** 모델.
- 보안·권한·자동화 안전장치(수동 메일 발송 등)를 갖춘 **책임 있는 AI 운영** 예시.
